# OpenClaw Life Coach 调料包整体计划清单

## 0. 项目目标

本项目的目标不是做一个只有提示词的“会聊天 AI”，而是做一个真正能在 OpenClaw（小龙虾）中运行的 **Life Coach 调料包**。

它最终应具备以下能力：
- 有稳定、专业的 life coach 人格
- 有清晰的 skills 与会话工作流
- 有长期记忆与自净化机制
- 有知识库沉淀与精准调用能力
- 有多模态接入能力
- 有安全边界与风险治理能力
- 能通过 OpenClaw workspace / skills / agents / tools / plugin 体系稳定运行

---

## 1. 总体设计原则

### 1.1 三层结构

本项目建议按三层来建设：

#### 第一层：Markdown 认知层
负责：
- 人格
- skill 说明
- 会话规则
- 提问框架
- 安全边界
- 知识块内容

对应文件：
- `workspace/prompts/*.md`
- `workspace/.agents/*.md`
- `workspace/skills/*/SKILL.md`
- `workspace/knowledge/**/*.md`
- `workspace/memories/*.md`

#### 第二层：配置与调度层
负责：
- OpenClaw agent 配置
- skills allowlist
- 环境变量注入
- 多模态模型配置
- 路由规则配置

对应文件：
- `config/openclaw.json.template`
- `install/openclaw-config.patch.example.json`
- `config/*.env.example`

#### 第三层：代码 / 插件执行层
负责：
- memory pipeline
- knowledge retrieval
- skill router
- safety router
- multimodal gateway adapter
- 数据结构化与状态管理

这一层后续会补代码，不应只停留在 markdown。

## 4. Markdown 与代码的实施边界

### 4.1 适合继续用 Markdown 实现的
- 人格
- 提问框架
- 对话策略
- 边界规范
- 知识块内容
- skill 的方法说明
- 风险与伦理规则

### 4.2 必须逐步代码化的
- memory 算法
- knowledge 检索
- skill 自动分流
- multimodal adapter
- 路由器
- 数据结构化
- 状态管理

结论：

> **Markdown 定义行为，代码保证执行。**
