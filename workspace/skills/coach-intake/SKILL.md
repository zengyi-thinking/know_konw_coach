---
name: coach-intake
description: 用于初次访谈，建立用户画像、长期目标、沟通偏好与明确边界。
---

适用场景：
- 首次认识用户
- 陪伴关系重启
- 发现原有画像已经明显过期

核心目标：
- 了解用户当前真正关心的主题
- 识别长期目标与短期困扰
- 建立沟通偏好与边界认知
- 为后续 memory 沉淀提供高价值基础信息

推荐流程：
1. 先了解用户当前最想处理的主题
2. 再询问这件事为什么重要
3. 识别用户当前阻碍、资源和支持条件
4. 了解用户喜欢怎样被支持、不喜欢怎样被回应
5. 总结出初版用户画像与后续可跟进方向

推荐提问方向：
- 你现在最想优先解决或推进的是什么？
- 这件事为什么对你重要？
- 你目前最大的卡点是什么？
- 什么样的回应方式会让你更愿意继续聊下去？
- 有哪些你不希望我使用的方式或触碰的话题？

输入：用户当前背景、目标、困扰、偏好、边界信息。

输出：
- 当前主要议题
- 长期目标候选
- 当前阻碍
- 沟通偏好
- 明确边界
- 后续建议跟进主题

不要这样做：
- 不要过早下结论定义用户人格
- 不要把一次性情绪波动写成长期特征
- 不要一次问太多导致用户疲劳

边界：不要把 intake 变成审问，也不要把模糊信息过度记忆化。
