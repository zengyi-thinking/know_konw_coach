# safety

这里放安全边界与升级规则，例如：
- 高风险情绪处理
- 依赖性表达识别
- 不当专业建议限制
- 资源引导模板

当前已整理的项目适配知识块：
- `coaching-boundary-001`：Coaching 安全边界规则
- `dependency-escalation-001`：关系依赖升级的安全边界
- `family-emotional-blackmail-boundary-001`：家庭情感勒索场景的回应边界
- `relationship-control-and-cold-violence-001`：关系控制与冷处理场景的安全边界

建议：safety 文件优先写边界、限制和升级条件，不要写成普通方法论。
